## 📋 테스트 케이스

### 테스트 1: [P0-A] 대화형 니즈 수집

**목표**: AI 챗봇이 필수 정보 4가지를 자동으로 수집하는지 확인

**단계**:
1. **Step 1**: 초기 니즈 입력
   ```
   입력: "신임 팀장들이 MZ세대와 소통하는 데 어려움"
   ```

2. **Step 1.5**: AI 챗봇 Q&A
   ```
   예상 AI 질문:
   - "팀장이 몇 명인가요?"
   - "가장 시급한 문제가 무엇인가요?"
   - "강의형 vs 실습형?"
   ```

3. **사용자 답변**:
   ```
   "20명이고, 특히 피드백할 때 어려워요. 실습 위주로"
   ```

4. **정보 완성도 표시**:
   - ✅ 교육 인원: 20명
   - ✅ Pain Point: 피드백
   - ✅ 실습 비중: 실습형
   - ⏳ 기존 시도: (선택사항)

5. **"검색 진행" 버튼 활성화**:
   - 필수 정보 3개 이상 완성 시 활성화됨

**성공 기준**:
- [ ] AI가 자동으로 부족한 정보에 대해 질문을 던짐
- [ ] 대화 히스토리가 누적되어 표시됨
- [ ] 정보 완성도가 정확하게 표시됨 (4개 메트릭)
- [ ] 필수 정보 3개 완성 시 "검색 진행" 버튼 활성화됨

### 테스트 2: [P0-B] 검색 결과 강제 주입

**목표**: Retrieved Modules JSON이 제대로 생성되고 커리큘럼에 반영되는지 확인

**단계**:
1. **Step 2 모듈 검색**:
   - "🟢 핵심 모듈 (8개)" 표시 확인
   - 각 모듈의 유사도(%) 표시 확인

2. **Step 3 제안서 생성**:
   - Streamlit이 "✍️ 최적 커리큘럼 조합 및 제안서 작성 중..."이라고 표시

3. **생성된 커리큘럼 분석**:
   ```
   아래 체크리스트 항목들을 수동으로 확인해야 함:
   ```

**성공 기준**:
- [ ] 생성된 커리큘럼에 실제 모듈명이 포함되어 있음
- [ ] 각 모듈의 세부 활동이 구체적으로 명시되어 있음
  - ❌ "피드백 스킬" (단순 모듈명)
  - ✅ "페어 피드백 롤플레이 / 피드백 진단지 작성" (구체적 활동)
- [ ] 시간 합계가 할당된 시간을 초과하지 않음 (P1-1)
- [ ] 모든 활동이 DB 모듈에서 추출된 콘텐츠인 것으로 보임
- [ ] AI가 임의로 창작한 활동이 없음

### 테스트 3: 전체 흐름 (E2E)

**시나리오**: 신입사원 영업 스킬 교육

**입력**:
```
산업군: 유통
교육대상: 신입사원
교육기간: 8H (1일)
고객사명: 이마트

Step 1 니즈: "신입사원들의 고객응대 스킬이 부족해요"
Step 1.5 Q&A:
  AI: "신입사원이 몇 명이고, 가장 큰 문제가 뭔가요?"
  User: "200명 정도인데, 특히 클레임 상황에서 대처를 못해요"
  AI: "강의 vs 실습 어느 쪽을 원하세요?"
  User: "실습으로, 실제 상황에서 대처하는 법을 배우고 싶어요"
```

**기대 결과**:
```
Step 2: 고객응대, 클레임 처리 관련 모듈 8개 검색
Step 3: 생성된 커리큘럼 예상 내용
  - 도입: 고객응대 마인드셋
  - 핵심 1: 클레임 상황 롤플레이
  - 핵심 2: 실제 대응 스크립트 실습
  - 마무리: 개인별 액션 플랜
  → 총 8H 정확히 구성
```

**성공 기준**:
- [ ] 200명이 언급된 니즈 문맥이 반영됨
- [ ] 클레임 처리 관련 구체적 활동 포함
- [ ] 실습형 방식으로 구성됨
- [ ] 모든 활동이 검증된 DB 모듈에서 온 것임
- [ ] 8시간 시간 구성이 정확함

## 🔍 상세 검증 항목

### A. 대화형 UI 검증 (P0-A)

| 항목 | 검증 방법 | 기준 |
|------|---------|------|
| 채팅 표시 | st.chat_message 사용 | 대화 풍선 형태로 표시됨 |
| 히스토리 누적 | 이전 메시지들이 계속 표시 | 대화 맥락 유지됨 |
| 정보 감지 | 키워드/정규식 검출 | 20명 → 교육인원 감지 |
| 버튼 활성화 | 조건부 활성화 | 정보 3개 이상 완성 시 활성화 |

### B. RAG 강제 주입 검증 (P0-B)

| 항목 | 검증 방법 | 기준 |
|------|---------|------|
| JSON 생성 | Retrieved Modules 구조 | 8개 모듈, rank/similarity 포함 |
| 프롬프트 주입 | app.py 소스 코드 검토 | retrieved_modules_json이 prompt에 포함 |
| 지시 명확성 | 프롬프트 텍스트 | "이 목록에 없는 활동을 창작하면 안 됩니다" |
| 컨텐츠 활용 | 생성된 커리큘럼 | 모든 활동이 JSON의 세부내용요약에서 추출됨 |

### C. 통합 검증 (P1-1 포함)

| 항목 | 검증 방법 | 기준 |
|------|---------|------|
| 시간 정합성 | Step 3에서 시간 메트릭 표시 | "✅ 시간 정합성 OK" 또는 경고 |
| 플레이스홀더 | [고객사명] 치환 | 이마트 → 모든 고객사명이 치환됨 |
| 다운로드 | Word/Markdown 파일 | 형식 정상, 내용 일관성 |

```
[P0-B] Step 2: 시간관리, 자기관리, 생산성 관련 모듈 검색

  → 피드백 롤플레이

  → 성과 진단 토의
  → 액션 플랜 수립
  → 실행 모니터링 설계
- 마무리: 개인별 실행 계획 (1.5H)
  → 체크리스트 작성
  → Q&A

✅ 모든 활동이 검증된 모듈에서 추출됨
✅ 총 8시간 정확히 구성
✅ 실습형 중심
```

## ❌ 문제 상황 시 체크리스트

### 챗봇이 계속 같은 질문을 할 때
```
확인 사항:
- 사용자 답변이 제대로 입력되었는가?
- 정보 감지 키워드가 포함되어 있는가?
- 예: "20명" (O) vs "20" (X, "명" 필요)
```

### 검색 결과가 나타나지 않을 때
```
확인 사항:
- 모듈 DB가 로드되었는가? (sidebar 확인)
- Gemini API 키가 유효한가?
- 니즈가 충분히 정제되었는가?
```

### 생성된 커리큘럼에 AI 창작이 포함되어 있을 때
```
이는 [P0-B] 프롬프트 지시가 제대로 작동하지 않은 것
→ 프롬프트의 [P0-B] 섹션 확인
→ "이 목록에 없는 활동을 창작하면 안 됩니다" 명확히 포함되어 있는가?
```
